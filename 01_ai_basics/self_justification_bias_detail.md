## 1. Purpose（目的）

自己正当化の理論的背景と、AI利用・AI開発における類似挙動のリスク、検出方法、統制方法を説明する。

先に [基礎](./self_justification_bias_basic.md) を読むと理解しやすい。

## 2. Terminology and Scope（用語と対象範囲）

### 2.1 用語

- **Self-Justification Bias（自己正当化バイアス）**
  - 過去の判断・行動・説明を誤りと認めず、後付けの理由で正当化し、誤った方針を維持しようとする傾向（人間）。
- **Cognitive Dissonance（認知的不協和）**
  - 矛盾する認知を同時に持つときに生じる心理的不快。低減のために行動を変えるか、信念・評価・説明を変える。
- **自己正当化に類似した出力挙動（本資料でのAI適用）**
  - 過去の出力や文脈との整合性を保つ過程で、誤った説明を維持・補強するように見える応答パターン。

### 2.2 必須注記（AI適用の制限）

本資料で扱うAIの「自己正当化」は、人間と同じ心理状態がAI内部に存在することを意味しない。過去の出力や与えられた文脈との整合性を保つ過程で、誤った説明を維持または補強するように見える出力挙動を、実務上理解しやすく表現したものである。

使用してよい表現 / 禁止表現：

| 使用可 | 禁止 |
|---|---|
| 自己正当化に類似した出力挙動 | AIは認知的不協和を感じる |
| 過去の回答との整合性を維持するような出力 | AIは自尊心を守ろうとする |
| 誤った前提を保持したまま理由を追加する | AIは非を認めたくない |
| 誤りの訂正より説明の再構成を優先するように見える | AIには人間と同じ自己正当化バイアスがある |
| 実務上、自己正当化と同様の結果を生じさせる | AIが意図的に自己防衛する |

### 2.3 対象範囲

- 対象：過去回答・設計判断の維持、後付け正当化、訂正回避に見える挙動と、その検出・防止・是正。
- 対象外の主題：自己帰属バイアスそのもの（比較説明のみ）、エージェント完全自動化の詳細実装（L4準備中）。

## 3. Self-Justification and Cognitive Dissonance（自己正当化と認知的不協和）

人間における自己正当化は、判断や行動と後から得た事実の矛盾に対し、誤りを認めて修正する代わりに、理由・評価・解釈を変えて「自分は正しかった」と維持する傾向として説明する。

認知的不協和の古典的な説明では、矛盾を低減するために、

1. 行動を変える（例：やめる）
2. 認知を変える（合理化・否認）
3. 新しい認知を足す（別の正当理由を加える）

といった経路がありうる。自己正当化は、とりわけ 2 と 3 によって過去の判断を守る方向に偏るときに問題になる。

また、説明生成が「正しさの追求」だけでなく、自己正当化や不協和の低減など方向づけられた動機を持ち得る、という枠組みも参考になる。これは**人間の説明行動**の議論であり、AI内部に同一の動機があることを示すものではない。

## 4. Distinction from Self-Attribution Bias（自己帰属バイアスとの区別）

| 項目 | 自己正当化バイアス | 自己帰属・自己奉仕的バイアス |
|---|---|---|
| 中心 | 過去の判断や行動を正しいまま維持する | 成功・失敗の原因をどこへ帰属するか |
| 典型例 | 誤りを認めず後付け理由を作る | 成功は自分、失敗は外部要因とする |
| 関連 | 認知的不協和、動機づけられた推論 | 原因帰属、自己奉仕的評価 |
| 今回の主題 | **対象** | 比較説明のみ |

旧テーマ名（Self-Attribution Bias）を、自己正当化の別名として扱ってはならない。

## 5. General Structure（自己正当化が発生する一般的構造）

```text
過去の判断・行動
        ↓
矛盾する事実の発見
        ↓
不整合が生じる
        ↓
A. 判断や行動を修正する
B. 説明を変えて過去の判断を正当化する
```

実務で危険なのは、B が選ばれ続け、正本・差分・テスト結果より「説明の流暢さ」が優先される場合である。

## 6. Similar Output Behaviors in AI（AIにおける類似した出力挙動）

観測・実務上よく問題になるパターン：

1. **正本と異なる過去回答の維持**
   - 相違を「表現の違い」「実質同じ」へ矮小化する。
2. **実装済み・commit済みを理由にした仕様変更**
   - 「拡張性のため必要だった」など、指示されていない理由を後付けする。
3. **未確認事項の補完とその正当化**
   - 「一般的なベストプラクティス」で正本違反を説明する。
4. **訂正要求への弁明優先**
   - 誤りの有無を確定する前に、長い理由説明へ進む。

これらは、**同じAIに作成と最終判定を独占させる**、**過去回答だけを根拠にする**、**仕様不一致でも実装を止めない**といった運用と組み合わさると被害が大きい。

## 7. Limits of Self-Correction（AIの自己訂正能力の限界）

「自己監視の限界」を主題にするのではなく、訂正能力の限界として最小限を述べる。

- 外部フィードバックなしの内在的自己訂正では、LLMが推論誤りを十分に直せない場合があり、自己訂正後に性能が低下する報告がある。
- 自己訂正手法は、学習時・生成時・事後訂正などに分類され、外部ツール・知識・別モデルからのフィードバックが重要になることが多い。

含意：

- 「もう一度自分で見直して」だけに依存しない。
- 正本、Git差分、テスト、別AI、人間レビューを組み合わせる。
- 自己訂正の可否と、自己正当化に類似した維持挙動は別問題として扱う（混同しない）。

## 8. Impact on SCAO Workflow（SCAO開発工程への影響）

SCAOの4AI役割への対応例：

| 役割 | 担当 | 自己正当化類似挙動への要点 |
|---|---|---|
| 調査 | Gemini | 一次情報・外部証拠を確認する |
| 設計 | ChatGPT | 過去判断と現在の証拠を分離して設計判断する |
| 実装 | Cursor | 実装済みであることを理由に仕様違反を正当化しない |
| 独立レビュー | Claude | 作成者とは独立した立場から監査する |
| 最終判断 | 人間 | 説明の流暢さではなく証拠で採否を決める |

必須原則：

1. 同じAIに作成と最終判定を独占させない
2. 過去回答ではなく正本を確認する
3. 説明可能性と正しさを混同しない
4. 流暢な理由を証拠として扱わない
5. 誤りの原因説明より先に、誤りの有無を確定する
6. 仕様不一致時は実装を停止する

AIを分けるだけでは不十分な場合がある。テスト・差分確認・人間判断を組み合わせる。

## 9. Detection Checklist（検出チェックリスト）

### 9.1 回答内容

- [ ] 過去の主張を維持することが目的化していないか
- [ ] 新しい証拠を正当に評価しているか
- [ ] 誤りを「表現の違い」として縮小していないか
- [ ] 指示されていない仕様を後付けしていないか
- [ ] 一般論で正本違反を正当化していないか
- [ ] 結論を変えるべき証拠を無視していないか

### 9.2 開発作業

- [ ] 実装済みであることを理由に仕様を変えていないか
- [ ] commit済みであることを理由に不備を維持していないか
- [ ] 自分が作成した成果物を自分だけで評価していないか
- [ ] 正本、Git差分、テスト結果を直接確認しているか
- [ ] 未確認事項を明示しているか

## 10. Prevention（防止策）

1. **正本優先**：設計書・仕様書・発行済み差分を開いて確認する。
2. **根拠の具体化**：ファイル名・節番号・コミット・テスト名を答えさせる。
3. **区分強制**：事実 / 推測 / 未確認を分けて書かせる。
4. **訂正優先プロンプト**：理由説明の前に「誤りか／正しいか」を確定させる。
5. **役割分離**：生成AIと評価AIを分ける。別チャットまたは別文脈で再評価する。
6. **機械的検証**：テスト、静的解析、差分確認を併用する。
7. **停止条件**：仕様不一致・正本未確認・破壊的操作前は実装を止める（人間確認ゲート）。

## 11. Remediation Procedure（誤りが発生した場合の是正手順）

次の4段階で止める。

1. **「間違いでした」と明確に認める**
2. **何を確認していなかったかを示す**
3. **どの時点で停止すべきだったかを示す**
4. **修正後の最小1ステップだけを提示する**

禁止事項：

- 長い弁明
- 誤りを単なる表現問題へ縮小する
- ユーザーへの責任転嫁
- 過去判断を守るための新仕様作成
- 訂正前に複数の修正作業へ進む

## 12. Summary（まとめ）

- 自己正当化は、矛盾する事実に対して過去の判断を理由づけで維持する傾向である。
- 自己帰属バイアスとは区別する。
- AIについては心理状態を断定せず、類似した出力挙動と実務リスクとして扱う。
- 自己訂正の限界があるため、外部証拠と独立レビューが必要である。
- 検出チェックリストと4段階是正で、誤った方針の維持を早期に止める。
