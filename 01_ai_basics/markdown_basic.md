## 1. Purpose（目的）

移行テンプレートや生成AIへの指示（プロンプト）を扱うために必要な、最小限のMarkdownを理解する。
Markdownの網羅的な仕様の習得は目的としない。

## 2. Target Audience（対象読者）

- Markdownをほとんど使ったことがない初学者
- 生成AIへの指示や設計ドキュメントをこれから書く人

## 3. Learning Goals（学習目標）

- 見出し・箇条書き・コードブロック・表などの基本記法を使い分けられる。
- 生成AIへ構造化して指示する理由を説明できる。
- 移行テンプレートの見出し構造を壊さずに使える。
- Markdownは目的ではなく、意思疎通の手段だと理解する。

## 4. Main Content（本文）

### 4.1 Markdownとは何か

Markdown（マークダウン）は、記号を使って文章に見出しや箇条書きなどの構造を付ける、軽量な書き方である。
特別なソフトは不要で、テキストエディタがあれば書ける。

Markdown自体を覚えることが目的ではない。
Markdownは、自分の考えを整理し、生成AIや他の人に正確に伝えるための「手段」である。

### 4.2 見出し

`#` の数で見出しの大きさが決まる。文書の構造（章立て）を示すときに使う。

```markdown
# 大見出し（H1）
## 中見出し（H2）
### 小見出し（H3）
```

### 4.3 箇条書き

順番が重要でない項目を並べるときに使う。

```markdown
- りんご
- みかん
- ぶどう
```

### 4.4 番号付きリスト

順番が重要な手順を示すときに使う。

```markdown
1. 材料を用意する
2. 手順どおりに作る
3. 完成を確認する
```

### 4.5 チェックリスト

やることの一覧や確認項目に使う。

```markdown
- [ ] まだ終わっていない項目
- [x] 終わった項目
```

### 4.6 コードブロック

コマンドやソースコードは、バッククォート3つで囲む。地の文とコードを区別するときに使う。

````markdown
```
npm install
```
````

言語名を付けると見やすくなる。

````markdown
```python
print("hello")
```
````

### 4.7 表

複数の項目を整理して比較するときに使う。

```markdown
| 項目 | 説明 |
|------|------|
| 名前 | 利用者の名前 |
| 年齢 | 利用者の年齢 |
```

### 4.8 引用

他の文章や注意点を引用として示すときに使う。

```markdown
> これは引用です。
```

### 4.9 記法の使い分け

同じ内容でも、目的に合った記法を選ぶと伝わりやすい。

- 文書の構造を分けたい → 見出し
- 並列の項目を並べたい → 箇条書き
- 順番のある手順を示したい → 番号付きリスト
- コマンドやコードを示したい → コードブロック
- 項目同士を比較したい → 表

### 4.10 生成AIへ構造化して指示する理由

Markdownで構造を付けると、生成AIとの意思疎通が正確になる。

- 目的・前提・制約などを見出しで分けると、生成AIが構造を理解しやすい。
- 箇条書きや番号付きリストにすると、指示内容の抜け漏れに気づきやすい。
- 出力形式を指定するときも、Markdownで例を示すと伝わりやすい。

つまりMarkdownは、生成AIに「読みやすく・誤解されにくく」指示するための手段である。

### 4.11 移行テンプレートの構造を壊さない注意

- 移行テンプレートの見出し（`## 1. 対象プロジェクト` など）は、番号や文言を勝手に変えない。
- 見出しを消したり順番を入れ替えたりしない。
- 各見出しの下に内容を書き足す形で使う。

## 5. Practical Checkpoints（実務上の確認点）

- 指示（プロンプト）や設計ドキュメントで、見出し・箇条書き・コードブロック・表を目的に応じて使い分けているか。
- テンプレートの見出し構造を崩していないか。
- 見た目を飾ることが目的になっていないか（伝わることが目的）。
